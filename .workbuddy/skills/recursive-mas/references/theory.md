## 1. 隐空间自回归生成

标准自回归解码中，last-layer 隐状态 hₜ 被投影到词汇空间预测下一个 token。隐空间生成则保持递归在连续表示空间中：

```
hₜ₊₁ = fθ([E≤t; hₜ])
```

新产生的 hₜ₊₁ 称为模型的 **ongoing latent thought**——不经过解码直接参与下一步推理。

### 与标准解码的区别

| | 标准解码 | 隐空间生成 |
|--|---------|-----------|
| 输出 | 离散 token | 连续向量 |
| 下一步输入 | Token embedding | 隐状态向量 |
| 词汇表投影 | 需要 | 不需要 |
| 语义保真 | 信息瓶颈（|V| 维选择） | 连续空间保留完整信息 |

## 2. 递归计算形式化

递归语言模型通过复用相同 Transformer 层栈增加推理深度：

```
H⁽⁰⁾ = E
H⁽ʳ⁾ = fθ(H⁽ʳ⁻¹⁾),  r = 1, ..., n
```

最终预测基于最后一轮 H⁽ⁿ⁾，经过 n 次递归精炼。

**关键洞察**：在 RecursiveMAS 中，每个 Agent 等价于一个 RLM 层，信息在 Agent 间以隐状态流的形式循环传递，整个系统构成一个递归环路。

## 3. 递归多智能体演化

**定义 2.1 (Recursive Multi-Agent Evolution)**:

系统 ℋ = {H₁, ..., Hₙ} 的递归演化是渐进精炼过程：

```
S⁽⁰⁾ →[Evolve]→ S⁽¹⁾ →[Evolve]→ ... →[Evolve]→ S⁽ⁿ⁾
```

每一步中，每个 Agent 通过：
1. 与自身推理状态的交互
2. 与其他 Agent 隐状态的交互

来调整其隐表示，使更新后的系统更好地对齐给定问题。

### 演化的两个维度

- **Intra-agent**: Agent 内部的 latent thoughts 自回归（Inner Link 驱动）
- **Inter-agent**: Agent 间的隐状态传递（Outer Link 驱动）

## 4. 运行时复杂度分析

### 命题 3.1 (RecursiveMAS Runtime Complexity)

设 N = Agent 数量, m = 每个 Agent 的 latent steps, t = 输入长度, dₕ = 隐维度, |V| = 词汇表大小。

**文本中介递归 MAS**:
```
Θ(N · (m|V|dₕ + (t+m)dₕ² + (t+m)²dₕ))
```

**RecursiveMAS (with RecursiveLink)**:
```
Θ(N · (mdₕ² + (t+m)dₕ² + (t+m)²dₕ))
```

### 差异来源

文本中介 MAS 每轮每个 Agent 都需要：
1. 解码隐状态到 token（|V|·dₕ 投影）
2. 重新编码 token 到嵌入

RecursiveMAS 通过 RecursiveLink 直接在隐空间转换，省去了词汇表投影的 |V|·dₕ 开销。在 |V| >> dₕ 的常见情况下（如 |V| = 32000, dₕ = 2048），加速比可达 15.6× 每步。

## 5. 梯度稳定性定理

### 定理 4.1 (Gradient Stability)

**前提**: token 置信度足够高（熵 ≤ ε, ε ≪ 1）

**文本中介递归** (ℛ_text):
```
‖∂ℛ_text(h)/∂h‖₂ ≤ O(ε) ≪ 1
```
→ 梯度消失，递归训练无法有效传播信号

**RecursiveLink** (ℛ):
```
‖∂ℛ(h)/∂h‖₂ ≥ Ω(1 - √(1/dₕ · log(1/δ)))
```
→ 梯度范数接近 1，递归训练信号稳定

### 直觉解释

文本中介的梯度路径需要经过：
```
隐状态 → softmax → 离散 token → embedding lookup → 隐状态
```
softmax 的高置信度输出接近 one-hot，对应的 Jacobian 几乎为零矩阵。

RecursiveLink 的梯度路径：
```
隐状态 → 线性变换 + 残差 → 隐状态
```
残差连接保证梯度直通，线性变换的 Jacobian 范数可控。

### 实际意义

- **文本 MAS 无法有效端到端训练**：必须依赖 prompt 优化或独立微调
- **RecursiveMAS 支持系统级端到端优化**：梯度沿完整递归路径有效传播
- 这解释了为什么仅训练 RecursiveLink（极少参数）就能取得显著提升

## 6. 缩放律与消融分析

### 递归深度缩放

- 轻量 RecursiveMAS（sub-1.5B agents）随递归深度增加呈清晰缩放趋势
- 递归轮次 n 从 1 到 4，准确率持续提升
- n=2 是性能/成本的最佳平衡点（论文推荐默认值）

### RecursiveLink 架构消融

| 变体 | 设计 | 效果 |
|------|------|------|
| 完整 RecursiveLink | 残差 + 双层投影 | 最优 |
| 去除残差 | 仅 W₂σ(W₁h) | 性能下降明显 |
| 单层投影 | 仅 W₁h | 维度映射不充分 |
| 线性插值 | αh + (1-α)ℛ(h) | 次于残差设计 |

### 语义分布分析

递归过程中，隐状态的语义分布逐步收敛：
- 前几轮：各 Agent 表示差异较大
- 后几轮：表示趋于一致，形成系统级共识
- 收敛速度与协作模式相关（Sequential 最快，Mixture 最慢）

### 训练成本

- Inner-Loop: 每个 Agent 独立训练，可并行，约 1-2 GPU 小时/Agent
- Outer-Loop: 系统级训练，需顺序前向，约 3-5 GPU 小时
- 总计：远低于全参数微调（Full FT 的 1/10 以下）
